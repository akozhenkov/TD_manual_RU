ГРУППИРОВАНИЕ ДАННЫХ КАЛИБРОВКИ

Это диалоговое окно открывается нажатием кнопку Присвоить номер группы калибровочным замерам в окне Данных калибровки.

Перед вычислением коэффициентов калибровки калибровочные измерения должны быть разделены на группы данных, имеющих одинаковый азимут и угол, но разный разворот вокруг оси.

Эти группы пронумерованы целыми числами, начиная с 1.
Данные с номером группы 0 отображаются в списке, но исключены из расчета коэффициентов.

Можно назначить номер группы для каждой строки данных калибровки вручную (в диалоге "Редактировать данные калибровки" ), либо TopoDroid может назначать номера групп автоматически.

Существуют два принципа автоматической группировки:

  1. TopoDroid: все 56 измерений группируются по четыре замера. Замеры до одной и той же точки должны попасть в одну группу;
  2. PocketTopo: по четыре замера для первых 16 измерений (четыре группы), после чего каждый из оставшихся 40 замеров в отдельной группе;

Оба способа выдают одинаковые калибровки, но при использовании способа TopoDroid приложение может расчитывать отклонение откалиброванных данных от ожидаемых значений, на большем количестве данных и создавать более репрезентативные распределения остаточных ошибок. Метод PocketTopo отличается от метода TopoDroid лишь только тем, что первые 16 измерений (по 4 для каждой грани куба) объединяются в группы, а остальные измерения обрабатываются по отдельноси, что в теории дает более равномерное распределение в пространстве.

Действия:

  • Сбросить группы: сбрасывает все номера групп на 0
  • OK: назначает номера групп в соответствии с выбранным правилом (делает это для строк без номера группы и не помеченых как удаленые)

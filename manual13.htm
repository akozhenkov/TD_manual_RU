[13] ИМПОРТ-ЭКСПОРТ

Когда вы закончите отрисовку, экспортируйте съёмку в вашем любимом формате в zip архив. Архив ZIP является полной резервной копией вашей съёмка и всех экспоротированных файлов. Вы можете использовать его, чтобы переместить съёмку в другой аппарат.
После того, как вы загрузили zip архив на ваш компьютер и распаковали его, вы можете дооформить карту пещеры в любой программе, к которой вы привыкли. Поддержка других форматов файлов пещерной топосъёмки при импорте может быть не полной.

ЭКСПОРТ ДАННЫХ ТОПОСЪЁМКИ

Therion
Данные топосъёмки экспортируются в Therion используя данные в формате "normal".
Единицы измерения данных в соответствии с настройками пользователя.
Названтя групп топосъёмки перечислены, но команда закоментирована.
Комментарии к пикетам и обозначения (фиксированные, покрашеные) предшествуют данным нитки хода.
Более того команда "input" для файлов th2 и описание карты для каждого scrap добавлены в конец файла, но и закоментированы.

Разбор и интерпретация синтаксиса Therion в TopoDroid довольно ограничены. В принцыпе только данные в формате "normal" могут быть прочитаны. Также поддерживаются следующие команды Therion: "units", "flag", "extend", "mark" и "station".
Фиксированные точки ("fix") анализируются, но не учитывается так как "cs" (система координат) не поддерживается.

Compass
Compass формат данных поддерживаются как на импорт так и на экспорт.
Единицы измерения 'футы' и 'градусы'.

Walls
Данные съёмки могут быть экспортированы в формат Walls.
Единицы измерения данных в соответствии с настройками пользователя.

Survex
Данные съёмки могут быть экспортированы в формат Survex.
Единицы измерения данных в соответствии с настройками пользователя.
Настройки:

  • конец строки может быть либо Unix либо Windows,
  • боковые замеры могут быть экспортированы с номером пикета ДО (такой же номер как у пикета ОТ с добавлением букаы)

VisualTopo
VisualTopo формат данных поддерживаются как на импорт так и на экспорт.
Единицы измерения 'метры' и 'градусы'.

PocketTopo
TopoDroid поддерживает формат PocketTopo как на импорт так и на экспорт.

Экспорт содержит только данные съёмки.
При экспорте названия пикетов с алфавитно-цифровыми символами должны быть преобразованы в числовые. В этом случае TopoDroid заменяет каждый символ двумя цифрами. Более точно 'a' преобразуется в '00', 'b' в '01' и т.д., 'A' в '50', 'B' в '51', и т.д.. Любой другой символ (т.е., алфавитно-цифровой) преобразуется в '99'.

При импорте, "outline" и "profile" абрисы импортрируються.
PocketTopo использует несколько цветов для дифференциации типов линий. Следовательно используется преобразование из цветов в типы линий therion_types. Линейные и точечные типы могут быть с определены в настройках.

Может случиться так, что импортированный абрис не выровнен на нитку хода. В этом случае вы можете сдвинуть абрис в окне Абрисов чтобы он совпал с ниткой хода.

cSurvey
Экспорт данных в cSurvey содержит только данные съёмки и не содержит абрисов.

PLT
Данные съёмки могут быть экспортированы в файл трека (OziExplorer) если вы задали координаты пикета.

KML
Данные съёмки могут быть экспортированы в файл KML (GoogleEarth) если вы задали координаты пикета.

CSV
Данные съёмки могут быть экспортированы как список с полями разделенными запятой:

  • пикет "от"
  • пикет "до"
  • длинна, азимут, угол
  • метка (флаг) если задан

Комментарии к пикетам не экспортируются. Краткий заголовок предваряет список данных.

DXF
Данные съёмки могут быть экспортированы как 3D DXF разделенный на слои

  • пикеты
  • нитка
  • боковые замеры

ЭКСПОРТ АБРИСОВ

Therion
Therion is the cave map drafting program of choice for TopoDroid.

Sketches are "natively" saved either as Therion files or in TopoDroid binary format.
The conversion from Therion to binary is "automatic": when you open a sketch and TopoDroid does not find the binary file, it looks for the Therion file.
The conversion from binary to Therion must be requested through the appropriate menu in the Survey window.

By default TopoDroid, unlike "xtherion", does not add an extra point coinciding with the first point, to closed area borders.

cSurvey
These exports contain both the plan and profile sketches, as well as the survey data. The sketch items are distributed on suitable cSurvey layers.

If your survey has only one sketch this is a complete export of your work (except for the cross-sections). If the survey has more than one sketch, you must export each sketch individually and merge the exports with cSurvey.

DXF
Sketches can be exported as 2D DXF files with layers for each point, line and area type in addition to the layers of the data 3D DXF.

The sketch items in the 2D DXF lie in the Z=0 plane.

SVG
Sketches can be exported as SVG files.

PNG
Image file in PNG format is used only to export sketches. The image resolution (pixels per meter) can be changed through a setting. If the system does not have enough memory to create the image at the set resolution, TopoDroid uses a smaller resolution until the export succeeds.

CALIBRATION EXPORT

Calibrations can be exported and imported as CSV files.

IMPORT/EXPORT SETTINGS

  • PocketTopo color-map: defines how to map PocketTopo colors on types of sketch items, for import of PocketTopo files.
  • Sketch auto-stations: whether to automatically add station points to the saved therion file(s). If you want to use the sketch files in therion they must have station points to define their scale. You can choose manually which point to add (by editing the sketch drawing), or let TopoDroid add the stations. The program will add all the stations that are "inside" the drawing (inside the drawing convex hull).
  • Data export format: defines the default format to which data are exported. This file is automatically generated and inserted in the ZIP export.
  • Station prefix: whether to add the survey prefix to the stations for the cSurvey export.
  • Survex line termination: either Unix or Windows
  • Survex named splay: whether to name the TO station of splays
  • Survex LRUD: whether to add LRUD lines to survex exports [default no]
  • PNG resolution: the resolution of the PNG files for sketch exports. It can be a number between 1 and 10. The higher the bigger the image (if you set it too high the system may not have enough memory to create the PNG).
  • PNG background color: you can set the background color of PNG images by specifying the RGB values (three integers between 0 and 255).
  • DXF scale: scale of the DXF export of sketches.
  • AutoCAD version for the DXF export: either 6 (AutoCAD R9) or 12 (AutoCAD R13). In version 6 DXF export bezier curves are replaced by polylines. Version 12 DXF export uses splines, but is still buggy.

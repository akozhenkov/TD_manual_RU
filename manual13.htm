[13] IMPORT EXPORT

When you are done, export the survey in your favourite format and make a zip archive. The zip archive is the a complete backup of your survey, and its exports. You can use it to transfer the survey to another android.
After you have downloaded the zip to your PC and unzipped it, you can complete the cave map with whatever program you are used to. The support of other cave survey file format in import may not be complete.

SURVEY DATA EXPORT

Therion
Survey data are exported in Therion format, using the data "normal" form.
Data units are according to the user options.
The team names are listed but the command is commented.
Station comments and marks (fixed, painted) preceed the centerline data.
Furthermore "input" of the th2 files and definitions for a map for each scrap are added at the end, but are commented.

TopoDroid parsing of Therion data syntax is very limited. Basically only data in "normal" form are understood. The Therion data commands "units", "flag", "extend", "mark" and "station" are supported.
Fixed points ("fix") are parsed but not taken into account as "cs" is not supported.

Compass
Compass data file are supported both in input and in export.
Data units are 'feet' and 'degrees'.

Walls
Survey data can be exported as Walls files.
Data units are according to the user options.

Survex
Survey data can be exported as Survex files.
Data units are according to the user options.
Settings:

  • line termination can be either Unix or Windows
  • splay shots can be exported with a named TO station (named as the FROM station with a letter appended)

VisualTopo
VisualTopo data file are supported both in input and in export.
Data units are 'meters' and 'degrees'.

PocketTopo
TopoDroid supports PocketTopo both in import and in export.

The exports contain only the survey data.
On export station names with alphabetic characters must be converted to integer number. In this case TopoDroid replaces each character with two digits. More precisely 'a' goes to '00', 'b' to '01' and so on, 'A' to '50', 'B' to '51', and so on. Every other character (ie, non-alphanumeric) goes to '99'.

On import, "outline" and "profile" sketches are imported.
PocketTopo uses only a few colors to differentiate line types. Therefore a map from colors to therion_types is used. The line and point types can be through a setting.

It may happen that the imported sketch is not aligned with the survey data. In this case you can shift the sketch in the Sketch window, to fit the centerline.

cSurvey
Data exports as cSurvey file contains only the survey data.

PLT
Survey data can be exported as track file (OziExplorer) if you have georeferenced a station.

KML
Survey data can be exported as KML file (GoogleEarth) if you have georeferenced a station.

CSV
Survey data can be esported as a list of records with comma-separated fields:

  • "from" station
  • "to" station
  • distance, azimuth, clino
  • if present the flag

Shot comments are not exported. A brief header is prepended to the data list.

DXF
Survey data are exported as 3D DXF with layers

  • stations
  • legs
  • splays

SKETCH EXPORT

Therion
Therion is the cave map drafting program of choice for TopoDroid.

Sketches are "natively" saved either as Therion files or in TopoDroid binary format.
The conversion from Therion to binary is "automatic": when you open a sketch and TopoDroid does not find the binary file, it looks for the Therion file.
The conversion from binary to Therion must be requested through the appropriate menu in the Survey window.

By default TopoDroid, unlike "xtherion", does not add an extra point coinciding with the first point, to closed area borders.

cSurvey
These exports contain both the plan and profile sketches, as well as the survey data. The sketch items are distributed on suitable cSurvey layers.

If your survey has only one sketch this is a complete export of your work (except for the cross-sections). If the survey has more than one sketch, you must export each sketch individually and merge the exports with cSurvey.

DXF
Sketches can be exported as 2D DXF files with layers for each point, line and area type in addition to the layers of the data 3D DXF.

The sketch items in the 2D DXF lie in the Z=0 plane.

SVG
Sketches can be exported as SVG files.

PNG
Image file in PNG format is used only to export sketches. The image resolution (pixels per meter) can be changed through a setting. If the system does not have enough memory to create the image at the set resolution, TopoDroid uses a smaller resolution until the export succeeds.

CALIBRATION EXPORT

Calibrations can be exported and imported as CSV files.

IMPORT/EXPORT SETTINGS

  • PocketTopo color-map: defines how to map PocketTopo colors on types of sketch items, for import of PocketTopo files.
  • Sketch auto-stations: whether to automatically add station points to the saved therion file(s). If you want to use the sketch files in therion they must have station points to define their scale. You can choose manually which point to add (by editing the sketch drawing), or let TopoDroid add the stations. The program will add all the stations that are "inside" the drawing (inside the drawing convex hull).
  • Data export format: defines the default format to which data are exported. This file is automatically generated and inserted in the ZIP export.
  • Station prefix: whether to add the survey prefix to the stations for the cSurvey export.
  • Survex line termination: either Unix or Windows
  • Survex named splay: whether to name the TO station of splays
  • Survex LRUD: whether to add LRUD lines to survex exports [default no]
  • PNG resolution: the resolution of the PNG files for sketch exports. It can be a number between 1 and 10. The higher the bigger the image (if you set it too high the system may not have enough memory to create the PNG).
  • PNG background color: you can set the background color of PNG images by specifying the RGB values (three integers between 0 and 255).
  • DXF scale: scale of the DXF export of sketches.
  • AutoCAD version for the DXF export: either 6 (AutoCAD R9) or 12 (AutoCAD R13). In version 6 DXF export bezier curves are replaced by polylines. Version 12 DXF export uses splines, but is still buggy.
